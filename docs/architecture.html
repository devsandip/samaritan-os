Data flow — one action's life

  1. Run Layer fires the capability on its trigger
  2. Capability runs with injected context, produces a result
  3. Policy decides: auto-complete, or escalate to Sandip
  4. If escalated, emit a standardized action item
  5. Action Center renders it from its schema & routes it
  6. Sandip approves / edits / rejects (or asks for more)
  7. Execute in mode: guided · assisted · automated
  8. Confirm back & audit — or fail & offer fallback

Execution modes — how far the agent takes it

GuidedAgent gives content + a deep link; Sandip does it. (WhatsApp)
AssistedAgent stages it; Sandip commits. (Gmail draft → send)
AutomatedAgent completes on approval. (File a Notion row)

Mode = capability × risk/reversibility × effort-vs-payoff. The human is the executor of last resort.

The three contracts

  • Trigger — how the Run Layer knows when to fire a capability.
  • Action-Item — the shared shape every capability emits; why the Action Center is universal.
  • Execution / Capability — how a capability requests an action from the registry.

A capability that speaks all three is a first-class citizen — drop it in, no core changes.
